
## 多次运行验证结果（validate_slots_graph_and_memory.py）

脚本连续运行 3 次，每次 12 轮、冷启动 N0=3 / N1=8，检查槽记忆、图扩展与记忆 vs 先验。

### 关键检查点

| 轮次 | 验证目标 |  Run 1 | Run 2 | Run 3 |
|------|----------|--------|--------|--------|
| 2 | pure_llm 下无记忆时行为 | 答「没有可用的记忆」 | 用先验给出公式 | 用先验给出公式 |
| 4 | 偏好槽记忆（Python） | ✅ 用户偏好：使用 Python 写算法 | ✅ 你喜欢用 **Python** 来写算法 | ✅ 你喜欢使用 Python 写算法 + 记忆引用 |
| 5 | 公式槽记忆 | ✅ 公式 LaTeX | ✅ 公式 LaTeX | ✅ 公式 + a≠0 |
| 9 | 多槽摘要（slots_and_graph） | ✅ 5 条记忆全列出 | ✅ 5 条全列出 | ✅ 按记忆逐条引用 |
| 10 | 图阶段下公式召回 | ✅ 仅公式 | ✅ 仅公式 | ✅ 仅公式 |
| 11 | 图阶段下偏好召回 | ✅ Python | ✅ Python | ✅ Python + 记忆引用 |
| 12 | 换说法问公式 | ✅ 公式 + a≠0 | ✅ 公式 LaTeX | ✅ 公式 + a≠0 |

### 结论

1. **槽记忆稳定**：3 次运行中，第 4、5、9、10、11、12 轮均正确召回「Python」偏好与求根公式，且多次出现与写入的结构化记忆一致或显式「记忆引用」表述。
2. **图扩展不破坏检索**：第 8 轮起进入 slots_and_graph 后，第 9～12 轮的多槽摘要与单条召回均正常，无漏检或错答。
3. **轮次 2 存在方差**：pure_llm 下未注入记忆时，有时答「没有可用的记忆」（Run 1），有时直接用模型先验给出公式（Run 2、3），属模型行为差异；说明「有/无记忆」的差异在后续轮次仍能通过召回体现。
4. **state_norm**：三次最终约 0.83～0.84，状态向量稳定。

综合多次运行可认为：**槽记忆、图扩展阶段检索、以及“回答引用记忆”在本脚本设定下表现一致、可复现。**

## 同一问题在概念启用前后的提示词类比（是否有压缩）

- **脚本**：`examples/compare_prompt_with_without_concept.py`。对同一 query 分别用 `get_context_for_query(query, use_concept=False)` 与 `use_concept=True` 得到两段「发送给 LLM 的提示词」，并对比槽数、字符数。
- **方法**：PNMS 新增只读方法 `get_context_for_query(query, system_prompt=..., use_concept=None)`，只做编码→检索→图扩展→（可选）概念 Augment→上下文构建，不更新状态、不写槽，返回 `(context_str, num_slots)`。

### 类比与结论

| 维度 | 概念未启用 (use_concept=False) | 概念启用后 (use_concept=True) |
|------|--------------------------------|-------------------------------|
| **流程** | query → 编码 → 槽检索 → 图扩展 → **不**做概念 Augment → 按检索/图顺序截断到 max_context_slots → 拼成提示词 | query → 编码 → 槽检索 → 图扩展 → **概念 Augment（按与 query 的相似度对槽重排）** → 同一截断与拼接 |
| **槽数** | 与检索+图扩展后的槽数一致，再截断到 max_context_slots | **相同**（概念只重排，不删槽） |
| **字符数** | 由 system_prompt + 入选槽的 content 决定 | **相同**（同一批槽、同一 max_context_slots，仅顺序不同） |
| **是否压缩** | - | **本实现无字符/槽数压缩**；概念阶段仅做「重排」，使更相关的记忆优先进入固定上下文预算，属于**语义上的优先压缩**（更少 token 内更多相关信号），而非长度缩短。 |

文档 §9 中的「压缩推理模式」指概念模块对**推理路径**的压缩（高频模式固化到小模块），不是对单次提示词字符串的截断；当前实现里概念 Augment 只改变槽的**顺序**，不改变提示词总长度。
